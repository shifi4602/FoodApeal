## 🎯 Project Overview

**FoodAppeal** is an online food shop platform. Customers can:

- 🛍️ Browse and filter products by category, price range, and description
- 🛒 Add items to a shopping basket and place orders
- 👤 Register / log in and view their order history
- 🗺️ Find restaurant branches on an interactive map
- 🍳 Explore cooking and hosting services
- 🎁 Browse gift options

**Admins** can:

- ➕ Add new products with image upload
- ✏️ Edit existing product details
- 📋 Manage orders and view all users

## ⚙️ Backend

The backend is a clean, layered **ASP.NET Core Web API** following the **separation of concerns** principle. Each layer has a single responsibility and communicates only with the layer directly below it.

```
Controller → Service → Repository → EF Core → SQL Server
```

---

### Architecture & Layers

```
backend/
├── Entities/         🗄️  Domain models (EF Core entities)
├── DTOs/             📦  Data Transfer Objects
├── Repositories/     🔌  Data access (EF Core + LINQ)
├── Services/         🧠  Business logic
└── WebApiShop/       🌐  ASP.NET Core app (Controllers, Middleware, Program.cs)
```

Each project is a separate class library with clearly defined interfaces — making layers swappable and testable in isolation.

### DTOs (Data Transfer Objects)

DTOs live in the `DTOs` project and are used to decouple the API contract from internal database entities. They prevent over-posting, hide sensitive fields, and shape the data for the client.

All DTOs are defined as C# **records** (immutable by default):

#### ProductDTO — returned to the client

```csharp
public record ProductDTO(
    int Id,
    int CategoryId,
    string Name,
    string Description,
    double Price,
    string ImageUrl,
    bool IsAvailable,
    string CategoryName      // 👈 flattened from Category navigation
);
```

#### PostProductDTO — sent by the client to create/update

```csharp
public record PostProductDTO(
    int Id,
    int CategoryId,
    string Name,
    string Description,
    double Price,
    string ImageUrl,
    bool IsAvailable = true  // 👈 default value
);
```

#### OrderDTO

```csharp
public record OrderDTO(
    int Id,
    int UserId,
    DateOnly OrderDate,
    double OrderSum,
    string Status,
    ICollection<OrderItemDTO> OrderItems
);
```

#### PageResponseDTO<T> — generic pagination wrapper

```csharp
public record PageResponseDTO<T>(
    List<T> Data,
    int TotalItems,
    int CurrentPage,
    int PageSize,
    bool HasPreviousPage,
    bool HasNextPage
);
```

> 📌 **Why two Product DTOs?** `ProductDTO` includes `CategoryName` (resolved server-side), while `PostProductDTO` only needs the `CategoryId`. This avoids sending unnecessary data in requests and keeps responses rich.

### Entity Framework Core

**EF Core** is used as the ORM. The `ApiDBContext` is defined in the `Repositories` project:

```csharp
public partial class ApiDBContext : DbContext
{
    public virtual DbSet<Category>  Categories { get; set; }
    public virtual DbSet<Order>     Orders { get; set; }
    public virtual DbSet<OrderItem> OrderItems { get; set; }
    public virtual DbSet<Product>   Products { get; set; }
    public virtual DbSet<Rating>    Ratings { get; set; }
    public virtual DbSet<User>      Users { get; set; }
}
```

Relationships are configured with the **Fluent API** in `OnModelCreating`:

```csharp
// Order → User (many-to-one)
entity.HasOne(d => d.User)
      .WithMany(p => p.Orders)
      .HasForeignKey(d => d.UserId)
      .HasConstraintName("FK_Orders_Users");

// OrderItem → Order and OrderItem → Product
entity.HasOne(d => d.Order)
      .WithMany(p => p.OrderItems)
      .HasForeignKey(d => d.OrderId)
      .HasConstraintName("FK_OrderItems_Orders");
```

The connection string is stored in `appsettings.json`:

```json
{
  "ConnectionStrings": {
    "School": "Data Source=YOUR_SERVER;Initial Catalog=foodAppealShop2;Integrated Security=True;Trust Server Certificate=True"
  }
}
```

### Repositories Layer

Each entity has a dedicated repository that implements an interface. This pattern makes the data access logic mockable in tests.

**Example — `IProductRepository`:**

```csharp
public interface IProductRepository
{
    Task<List<Product>> GetProducts(int position, int skip, int?[] categoryIds,
        string? description, int? maxPrice, int? minPrice);
    Task<Product> GetProductById(int id);
    Task<Product> AddProduct(Product product);
    Task UpdateProduct(int id, Product product);
}
```

**Example — `ProductRepository` implementation with filtering & pagination:**

```csharp
public async Task<List<Product>> GetProducts(int position, int skip, int?[] categoryIds,
    string? description, int? maxPrice, int? minPrice)
{
    var query = _apiDbContext.Products
        .Where(product =>
            product.IsAvailable &&
            (description == null || product.Description.Contains(description)) &&
            (maxPrice == null || product.Price <= maxPrice) &&
            (minPrice == null || product.Price >= minPrice) &&
            (categoryIds.Length == 0 || categoryIds.Contains(product.CategoryId)))
        .OrderBy(product => product.Price);

    return await query
        .Skip((position - 1) * skip)
        .Take(skip)
        .Include(product => product.Category)  // 👈 eager loading
        .ToListAsync();
}
```

> 🔍 This single query handles **filtering by category, description, price range** AND **pagination** — all translated by EF Core into a single optimized SQL query.

### Services Layer

Services contain the **business logic**. They receive DTOs from controllers, interact with repositories, and use AutoMapper to convert between entity and DTO types.

**Example — `UserService`:**

```csharp
public async Task<UserDTO> AddUser(PostUserDTO user)
{
    // Map DTO → Entity, save, Map Entity → DTO
    return _mapper.Map<User, UserDTO>(
        await _userRepository.AddUser(
            _mapper.Map<PostUserDTO, User>(user)
        )
    );
}

public async Task<UserDTO> Login(LoginUserDTO loginUser)
{
    return _mapper.Map<User, UserDTO>(
        await _userRepository.Login(loginUser.Email, loginUser.Password)
    );
}

public bool IsPasswordStrong(string password)
{
    int passScore = _passwordService.GetPasswordScore(password);
    return passScore >= MinimumPasswordScore;  // score >= 2 required
}
```

All services are registered as **scoped** dependencies in `Program.cs`:

```csharp
builder.Services.AddScoped<IUserService, UserServices>();
builder.Services.AddScoped<IProductService, ProductService>();
builder.Services.AddScoped<IOrderService, OrderService>();
builder.Services.AddScoped<ICategoryService, CategoryService>();
builder.Services.AddScoped<IRatingService, RatingService>();
builder.Services.AddScoped<IPasswordService, PasswordService>();
```

### Controllers (API Layer)

Controllers are thin — they delegate all logic to services and only handle HTTP concerns (routing, status codes, request/response shaping).

**Example — `ProductsController`:**

```csharp
[Route("api/[controller]")]
[ApiController]
public class ProductsController : ControllerBase
{
    // GET /api/products?categoryIds=1&categoryIds=2&description=pizza&minPrice=5&maxPrice=50&skip=8&position=1
    [HttpGet]
    public async Task<ActionResult<List<ProductDTO>>> Get(
        [FromQuery] int?[] categoryIds,
        string? description,
        int? maxPrice,
        int? minPrice,
        int skip = 8,
        int position = 1)
    {
        var response = await _productService.GetProducts(position, skip, categoryIds, description, maxPrice, minPrice);
        if (response.Count > 0) return Ok(response);
        return NoContent();
    }

    // POST /api/products
    [HttpPost]
    public async Task<ActionResult<ProductDTO>> Post([FromBody] PostProductDTO newProduct)
    {
        ProductDTO returned = await _productService.AddProduct(newProduct);
        if (returned == null) return BadRequest();
        return CreatedAtAction(nameof(Get), new { id = returned.Id }, returned);  // 201
    }
}
```

| Controller           | Endpoints                                      |
|----------------------|------------------------------------------------|
| 🛒 `ProductsController`  | GET (all/filter), GET (by id), POST, PUT   |
| 🗂️ `CategoriesController`| GET all                                    |
| 📦 `OrdersController`    | GET (all), GET (by id), POST, PUT          |
| 👤 `UsersController`     | GET (all), GET (by id), POST, POST /login  |
| 🔑 `PasswordsController` | POST (validate strength)                   |
| 🖼️ `UploadController`    | POST /upload (multipart file)              |

### AutoMapper

AutoMapper is configured in `WebApiShop/AutoMapper.cs` as a `Profile`. It defines all bidirectional mappings between entities and DTOs:

```csharp
public class AutoMapper : Profile
{
    public AutoMapper()
    {
        CreateMap<Product, ProductDTO>().ReverseMap();
        CreateMap<Product, PostProductDTO>().ReverseMap();
        CreateMap<Order, OrderDTO>().ReverseMap();
        CreateMap<OrderItem, OrderItemDTO>().ReverseMap();
        CreateMap<User, UserDTO>().ReverseMap();
        CreateMap<User, PostUserDTO>().ReverseMap();
        CreateMap<User, UpdateUserDTO>().ReverseMap();
        CreateMap<User, LoginUserDTO>().ReverseMap();
        CreateMap<Category, CategoryDTO>().ReverseMap();
    }
}
```

> 💡 `.ReverseMap()` allows the mapping to work in both directions — entity → DTO and DTO → entity — with a single declaration.

### Middleware

Two custom middleware components are registered in the pipeline:

#### 🛡️ ErrorHandlingMiddleware

Wraps every request in a `try/catch`. If an unhandled exception occurs anywhere in the pipeline, it catches it, returns `HTTP 500`, and logs the full stack trace with NLog:

```csharp
public async Task Invoke(HttpContext httpContext)
{
    try
    {
        await _next(httpContext);
    }
    catch (Exception ex)
    {
        httpContext.Response.StatusCode = 500;
        _logger.LogError(ex + " call stack: " + ex.StackTrace);
    }
}
```

#### 📈 RatingMiddleware

Logs every incoming HTTP request to the `Ratings` table before passing it to the next handler — providing a complete request history:

```csharp
public async Task Invoke(HttpContext httpContext, IRatingService ratingService)
{
    var rating = new Rating
    {
        Host       = httpContext.Request.Host.Value,
        Method     = httpContext.Request.Method,
        Path       = httpContext.Request.Path,
        Referer    = httpContext.Request.Headers.Referer,
        UserAgent  = httpContext.Request.Headers.UserAgent,
        RecordDate = DateTime.Now
    };
    try { await ratingService.AddRating(rating); } catch { /* DB unavailable, continue */ }
    await _next(httpContext);
}
```

**Middleware pipeline order in `Program.cs`:**

```csharp
app.UseCors("AllowAngular");
app.UseHttpsRedirection();
app.UseErrorHandling();   // 🛡️ global error handler
app.UseRating();          // 📈 request logger
app.UseStaticFiles();
app.UseAuthorization();
app.MapControllers();
```

### Password Validation

Password strength is validated using the [**Zxcvbn**](https://github.com/zxcvbn-ts/zxcvbn) library — the same algorithm used by Dropbox. It returns a score from `0` (very weak) to `4` (very strong):

```csharp
public class PasswordService : IPasswordService
{
    public int GetPasswordScore(string password)
    {
        var result = Zxcvbn.Core.EvaluatePassword(password);
        return result.Score;  // 0–4
    }
}
```

In `UserService`, a minimum score of `2` is required to register:

```csharp
private const int MinimumPasswordScore = 2;

public bool IsPasswordStrong(string password)
{
    return _passwordService.GetPasswordScore(password) >= MinimumPasswordScore;
}
```

If the password is too weak, the controller returns `400 Bad Request` with a descriptive message.

### File Upload

The `UploadController` handles product image uploads:

```csharp
[HttpPost("upload")]
[Consumes("multipart/form-data")]
public async Task<IActionResult> Upload(IFormFile file)
{
    var fileName = Guid.NewGuid() + Path.GetExtension(file.FileName); // unique name
    var folderPath = Path.Combine(Directory.GetCurrentDirectory(), "wwwroot", "images");

    Directory.CreateDirectory(folderPath); // ensure folder exists

    var filePath = Path.Combine(folderPath, fileName);
    using var stream = new FileStream(filePath, FileMode.Create);
    await file.CopyToAsync(stream);

    var url = $"{Request.Scheme}://{Request.Host}/images/{fileName}";
    return Ok(new { url });  // returns the public image URL
}
```

Images are saved to `wwwroot/images/` and served as static files. The returned URL is stored in the product's `ImageUrl` field.

### State Management with Signals

The app uses Angular's built-in **Signals** for reactive state instead of a third-party state library:

```typescript
@Injectable({ providedIn: 'root' })
export class ProductsService {
    private productsSignal = signal<Product[]>([]);

    getProducts() {
        return this.productsSignal.asReadonly();
    }
}
```

```typescript
@Injectable({ providedIn: 'root' })
export class UserService {
    private currentUserSignal = signal<User | null>(null);

    getCurrentUser() {
        return this.currentUserSignal.asReadonly();
    }
}
```

Components read from signals directly in templates — Angular automatically re-renders only when the signal value changes.

### Route Guards

The `adminGuard` protects admin-only routes. It reads the current user from `UserService` and redirects to `/login` if the user is not an admin:

```typescript
export const adminGuard: CanActivateFn = () => {
    const userService = inject(UserService);
    const router = inject(Router);

    const user = userService.getCurrentUser()();
    if (user?.isAdmin) return true;

    router.navigate(['/login']);
    return false;
};
```

Applied to admin routes:


## 🗄️ Database Schema

```
Users
  └──< Orders
          └──< OrderItems >──── Products
                                   └── Category

Ratings  (standalone — records all HTTP requests)
```

| Table        | Key Relationships                               |
|--------------|-------------------------------------------------|
| `Users`      | Has many `Orders`                               |
| `Orders`     | Belongs to `Users`, has many `OrderItems`       |
| `OrderItems` | Belongs to `Orders` and `Products`              |
| `Products`   | Belongs to `Category`, has many `OrderItems`    |
| `Categories` | Has many `Products`                             |
| `Ratings`    | Standalone — one row per HTTP request           |
